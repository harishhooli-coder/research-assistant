# API documentation

The FastAPI app in `api/` is the **source of truth** for the HTTP contract.
This folder holds generated artifacts consumed by the web frontend and CI.

## Files

| File | Generated by |
|------|----------------|
| `openapi.json` | `scripts/export_openapi.py` |
| `../web/src/lib/api.generated.ts` | `openapi-typescript` (see `web/package.json`) |

## Regenerate locally

**Windows (PowerShell):**

```powershell
.\scripts\update-docs.ps1
```

**Any platform:**

```bash
python scripts/update_docs.py
```

Or step by step:

```bash
python scripts/export_openapi.py
cd web && npm run generate:api-types
```

Commit `docs/api/openapi.json` and `web/src/lib/api.generated.ts` when they change.

## CI

The [Docs workflow](../../.github/workflows/docs.yml) runs `scripts/check_docs.py` on
pull requests and pushes that touch `api/`, `docs/`, or `web/src/lib/`. The job fails
if generated files drift from the FastAPI schema.

## Interactive docs

When the API is running locally:

- Swagger UI: http://localhost:8000/docs
- OpenAPI JSON: http://localhost:8000/openapi.json

SSE event shapes (`progress`, `completed`, `failed`) are documented in the root
[README](../../README.md#api-contract) because they are not part of the OpenAPI schema.
